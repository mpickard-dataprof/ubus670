Behind the Curtain

What is AI Studio?

What is Google AI Studio?

Definition

Google AI Studio is a browser-based tool that lets you configure, test, and prototype with Google's Gemini models. It exposes the parameters and settings that consumer chatbots like Gemini Chat hide from you.

Gemini Chat — The Showroom

  • Simple conversational interface
  • Pre-set parameters (hidden)
  • No cost visibility
  • No system prompt access
  • Great for quick tasks

AI Studio — The Workshop

  • Full parameter controls
  • Custom system prompts
  • Real-time token counter
  • Model comparison tools
  • Export to code / API

The AI Studio Interface Tour

Conceptual layout — actual interface may vary. System instructions sit at the top of the central workspace; run settings (parameters) are on the right.

Why Configurers > Users

Day 5 moves you from User to Configurer. You won't need to write code, but you will control the settings that determine how AI behaves in your organization.

Section 2

The Dials

Model Parameters

Temperature: The Creativity Dial

Definition

Temperature controls how random or deterministic the model's output is. Low temperature = more predictable; high temperature = more creative and varied.

When to Use Which Temperature

Low
0.0 – 0.3

Classification & triage
Email sorting, ticket categorization, data extraction, sentiment analysis

Medium
0.4 – 0.7

Customer responses
Drafting replies, summaries, recommendations, professional writing

High
0.8 – 1.0

Brainstorming
Marketing copy, product naming, creative ideation, taglines

Rule of Thumb

The more you need consistency, the lower the temperature. The more you need variety, the higher.

Top-p and Top-k: Fine-Tuning Randomness

For MBA Purposes

Temperature is the big knob. Top-p (nucleus sampling) and top-k are fine adjustments. Nucleus sampling means the model only considers the smallest set of tokens whose combined probability exceeds p. You rarely need to change these, but understanding what they do makes you a better configurer.

Max Output Tokens

Definition

Max Output Tokens sets a ceiling on how long the model's response can be. One token is roughly 3/4 of a word in English. This parameter controls response length and cost.

Parameter Interaction: Discussion

Think about how these parameters interact:

Scenario A

Temperature = 0 and you ask the model to write a creative story.

What happens? What kind of story do you get?

Scenario B

Temperature = 2.0 and you ask the model to draft a legal contract clause.

What happens? Would you trust this output?

The Insight

Parameters and prompts interact. A perfect prompt with wrong parameters still produces wrong results. Configuration is as important as the question you ask.

The Business Takeaway

Parameters aren't just technical settings — they're business decisions.

Wrong Parameter Example

Beacon sets temperature to 1.2 for customer complaint classification. Result:

  • Same email classified as "Complaint" one time, "Inquiry" the next
  • Inconsistent routing breaks SLA promises
  • Angry customers, confused support staff

Right Parameter Choice

Temperature set to 0.1 for classification:

  • Same input produces same classification every time
  • Reliable routing, predictable SLAs
  • Trust in the system builds over time

Section 3

The AI Job Description

System Prompts

What is a System Prompt?

Definition

A system prompt is a set of persistent instructions that shapes every response the AI generates. Unlike a regular prompt that is sent once, the system prompt is active for the entire conversation.

Think of it as the Day 2 RCTFC framework made persistent:

  • Role → System prompt Role section
  • Context → System prompt Rules & Boundaries
  • Task → Defined once in system instructions, applied to every email
  • Format → System prompt Format section (JSON, categories, etc.)
  • Constraints → System prompt Escalation, Tone & Boundaries

Instead of writing RCTFC every time, you set it once and the system prompt applies it to every conversation.

Anatomy of a Great System Prompt

System Prompt: Beacon Email Triage

# Beacon Retail Group — Email Triage System Prompt ## Role You are Beacon Retail Group's email triage specialist. You classify incoming customer emails and draft responses. ## Rules - Classify every email into exactly ONE category: Complaint, Product Question, Order Status, Return/Exchange, Billing, Partnership Inquiry, Feedback/Praise - Assign a priority: Low, Medium, High, Critical ## Format Always respond with: Category: [category] Priority: [priority] Summary: [1-sentence summary] Suggested Response: [draft reply] ## Escalation - Legal threats → flag for Legal team - Safety concerns → flag for Safety Manager - Requests over $500 → flag for Supervisor ## Tone Professional, empathetic, solution-oriented. Use the customer's name when available. ## Boundaries - Never promise specific refund amounts - Never share internal pricing or margins - Never make up product information

Good vs. Bad System Prompts

Bad: Vague and Generic
You are a helpful assistant.
Answer customer emails politely.

Result: Inconsistent format, no classification, varies wildly between runs, no escalation rules, may share information it shouldn't.

Good: Detailed Beacon Prompt
[The full prompt from previous slide]
Role + Rules + Format +
Escalation + Tone + Boundaries

Result: Structured output every time, clear categories, automatic escalation, safe boundaries, consistent tone.

A vague system prompt is like a job description that says "do stuff." The more specific you are, the more reliable the AI becomes.

System Prompts as Governance

A well-crafted system prompt is your first line of AI governance. It defines what AI can and cannot do before any code is written.

Scope Control

What topics the AI can discuss and what it must decline

Escalation Paths

When to route to a human, which team gets flagged

Data Protection

How to handle sensitive info: PII, financials, legal matters

Looking Ahead

System prompts are governance you can write today. In Day 6, we'll test how well they hold up under adversarial conditions (red-teaming).

Iterating System Prompts

System prompts are living documents. Version them, test them, and improve them over time.

Iteration Process

  1. Write your initial system prompt
  2. Test with typical inputs
  3. Test edge cases (see right)
  4. Refine based on failures
  5. Version each iteration (v1, v2, v3...)

Edge Cases to Test

  • Email written in Spanish?
  • Profanity or abusive language?
  • Products Beacon doesn't sell?
  • Request for competitor pricing?
  • Blank email or just an attachment?
  • Multiple issues in one email?

Common Mistake

Writing a system prompt once and never testing it. Every prompt has blind spots — edge case testing reveals them before your customers do.

Section 4

What Does It Cost?

Token Economics

What Are Tokens? (Refresher)

Quick Callback to Day 1

Tokens are the units AI models use to process text. A token is roughly 3/4 of an English word. The word "Hello" is 1 token. "Unbelievable" might be 3 tokens (un + believ + able).

Token Pricing: Model Tiers

Disclaimer

Prices shown are illustrative and based on approximate early 2026 rates. Actual rates change frequently. Always check current pricing before deployment.

Gemini 3 Flash — The Toyota

Fast, affordable, great for most tasks

Input ~$0.50 / 1M tokens
Output ~$3.00 / 1M tokens

Gemini 3 Pro — The Mercedes

More capable, better reasoning

Input ~$2.00 / 1M tokens
Output ~$12.00 / 1M tokens

Pro costs ~4x more per input token and ~4x more per output token than Flash. Choose based on task difficulty.

The Beacon Email Math

Let's calculate a real deployment cost for Beacon's email triage:

Volume Estimate

200 emails/day × 30 days = 6,000/month

Input per email: ~700 tokens
(400 system prompt + 300 email)

Output per email: ~300 tokens

Monthly input: 6,000 × 700 = 4.2M tokens

Monthly output: 6,000 × 300 = 1.8M tokens

Monthly Cost

Gemini 3 Flash:

Input: 4.2M × $0.50/1M = $2.10

Output: 1.8M × $3.00/1M = $5.40

Total: ~$7.50/month

Gemini 3 Pro:

Input: 4.2M × $2.00/1M = $8.40

Output: 1.8M × $12.00/1M = $21.60

Total: ~$30.00/month

Illustrative pricing. Actual costs vary by model version, region, and pricing tier.

Cost vs. Value

Important Context

These ROI figures are illustrative. With Flash at ~$7.50/month vs. $600/month manual cost, that's roughly 80x savings. But real deployments have additional costs: development time, testing, monitoring, and human oversight. AI triage augments human work — it doesn't eliminate it.
