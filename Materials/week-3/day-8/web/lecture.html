The Business Problem

Beacon Retail Group has received 30 resumes for seasonal retail associate positions across 12 hiring locations. HR needs help screening candidates quickly and consistently — before the May hiring deadline.

Your mission: build an AI-powered screening pipeline that can:

  • Parse unstructured resumes into structured data
  • Evaluate each candidate against the job requirements
  • Rank all candidates and flag concerns

This isn't just an exercise — it's how real companies are starting to use AI in hiring.

Output: Ranked list + Flags

Key Insight: Each agent has ONE job. The Parser doesn't judge quality. The Evaluator doesn't parse resumes. The Ranker doesn't re-evaluate — it compares.

The Data Contract

The #1 failure mode is agents that talk past each other. Define what data flows between them:

// Parser Output → Evaluator Input { "candidate_id": "C-12", "name": "Maria Chen", "experience": [ { "title": "Assistant Store Manager", "company": "Lakeview Home", "years": 3.5 } ], "education": "AAS Business Admin", "skills": ["POS", "inventory", "Mandarin"], "availability": "Full-time May-Aug, all shifts", "certifications": ["CPR/First Aid"] }

This is a suggested schema. Your team can modify it — but Parser output MUST match what Evaluator expects. Agree on the contract before building.

Pitfalls & Pro Tips

⚠ JSON Inconsistency
Your Parser must output the SAME structure every time. Use low temperature (0.1-0.3) and include a JSON template in the prompt.
⚠ Missing the ID
Each resume starts with [C-XX]. Your Parser MUST carry this ID through. Without it, your submission can't be scored.
✅ Start Simple
Get a basic pipeline working first. A simple Evaluator that checks 3 criteria beats a sophisticated one that crashes.
✅ Test Incrementally
Don't run all 20 practice resumes at once. Test with 3-5 first, verify output, then scale up.
